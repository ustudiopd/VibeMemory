# Simple 100 Agent — 제품 소개서 v0.2 (S100 토큰 업데이트)

*업데이트 일자: 2025-08-16*

---

## 0\) 전체 개념 설명 (더 쉽게, 더 디테일하게)

**핵심 원칙(메인 메시지)**

1. **넣기만 하면 → 전처리·계획·업무가 자동화**  
   폴더에 던지기/메일 연동만 하면 시스템이 표준화·요약·추출을 수행하고 **마감 역산 일정/할 일**을 자동으로 제안합니다.

2. **S100 토큰 표준 → 외부/내부 AI의 입·출력을 유연 혼용**  
   모든 데이터가 **S100 토큰** 포맷으로 정리되어 **ChatGPT·Claude·Gemini 같은 외부 브라우저형 AI**도, **심플100 내부 AI**도 자유롭게 섞어 쓸 수 있습니다. 입력은 토큰에서 바로 만들고, 출력은 간단한 \*\*결과 규칙(결과 계약)\*\*대로 받아 **다시 토큰화**되어 잃지 않습니다.

**우리가 하는 일(한 줄):** “새 플랫폼으로 이주” 대신 **당신의 폴더/메일 흐름에 AI가 맞추게** 하여, 자료→표준화→계획/생성→보고/지식화를 자동으로 굴립니다.

### 외부 AI vs 내부 AI — 어떻게 혼용되나?

* **외부(브라우저형) AI:** ChatGPT/Claude/Gemini 등 웹에서 직접 사용. 심플100이 제공하는 *프롬프트 카드/간단 양식*을 **복사→붙여넣기**하거나, 필요한 경우 **작은 JSON 파일**로 **업로드(가능 시)** 합니다. 모델이 만들어준 결과도 **복붙/파일로 가져와** 심플100에 넣으면 곧바로 **토큰**으로 정리됩니다.

* **내부 AI:** 보안/비용/속도 요구가 높을 때 심플100 내부에서 바로 호출. 입·출력은 **S100 토큰/결과 계약**으로 동일하므로 UX는 같습니다.

* **혼용 라우팅:** 작업 종류·민감도·비용/성능 기준에 따라 **외부/내부 중 최적 경로**를 추천합니다(필요 시 사용자가 수동 선택).

### 오늘 바로 써보면 (Day‑0 경험) (Day‑0 경험)

1. **프로젝트 폴더 생성** → Gmail/Outlook/Drive/Dropbox **커넥터 ON**

2. 브리프·계약·레퍼런스를 **그냥 넣기**(Drag\&Drop). 메일/회의록은 자동 수집

3. 모든 항목이 **S100 토큰**(표준 데이터 단위)으로 정리 → \*\*중복 제거/해시/출처(프로비넌스)\*\*까지 붙음

4. **마감일 D**를 입력하면 **역산 일정/체크리스트/담당자** 자동 제안(±2\~3일 미세조정)

5. **외부 AI 라우팅**: 카피/콘티/리스크표 등 각 작업을 **가성비·성능·보안 기준**에 맞는 모델로 자동 추천/실행

6. \*\*결과 계약(JSON)\*\*으로 산출물을 받아 **artifact/checklist/task 토큰**에 즉시 반영

7. 한 화면에서 **상태·리스크·의존성**을 추적, 납품 후 **포스트모템**과 **교훈/금지사항**이 자동 축적되어 다음 프로젝트에 재사용

### 자동으로 해주는 것

* 파일/메일/회의록 **파싱·OCR·요약·개체/날짜 추출** → **표준화(S100 토큰)**

* **타임라인/간트** 초안, **체크리스트/담당/의존성** 제안

* **외부 AI BYO‑모델 라우팅**: 작업 종류·민감도·비용/성능 지표에 따라 최적 모델 추천/실행

* **결과 계약**으로 입·출력을 **토큰**에 재흡수(분실·사일로 방지)

* **상태 리포트/리스크 시그널**·지연 경보·스프린트 롤업

* **포스트모템**(성과지표·교훈·금지사항) 자동 생성 및 **지식화**

### 일부러 하지 않는 것

* 메신저·문서 에디터를 **새로 강요하지 않음**(현행 도구 존중)

* **벤더 락인** 유도 대신, **표준 토큰+결과 계약**으로 **느슨하지만 강한 결합**

* 승인되지 않은 데이터는 **읽지 않음**(프로젝트 단위 RBAC/감사/보존·파기/DSR 기본 탑재)

### 데이터 흐름 한 장

**넣기(폴더/커넥터)** → **정규화(S100 토큰)** → **계획(역산)** → **생성(외부 AI 라우팅)** → **재흡수(결과 계약→토큰화)** → **리포트/지식화**

한 문장 포지셔닝: 플랫폼 이주 없이, **폴더 하나**로 시작하는 **AI 프로젝트 운영 컨트롤 플레인**

### 실전 예제: ChatGPT 브라우저로 제안서 작성 — S100 토큰 입·출력

**핵심 개념(코드 없이 쉽게):**

* **S100 토큰 \= 범용 어댑터(택배 상자).** 자료와 결과를 같은 규격으로 담습니다.

* **ChatGPT/Claude/Gemini \= 외부 엔진.** 브라우저에서 직접 써도, 결과만 상자(S100 토큰)에 담아 오면 심플100이 자동으로 프로젝트에 꽂습니다.

* 이 구조가 **원칙 1**(넣기만 하면 자동화) \+ **원칙 2**(BYO‑AI 유연 혼용)를 동시에 실현합니다.

#### *Step 1\) 준비 — 폴더와 메일만 연결*

* RFP PDF와 Q\&A 메일 스레드가 있는 **프로젝트 폴더**를 지정하고, **Gmail/Outlook 커넥터**를 켭니다.

* “제안서 초안 만들기”를 누르면 **프롬프트 카드**(목표/참고자료/원하는 결과 형태)가 생성됩니다.

#### *Step 2\) (외부 브라우저에서) ChatGPT로 초안 만들기*

* 프롬프트 카드의 \*\*“복사하기”\*\*를 눌러 **ChatGPT 브라우저**에 붙여 넣습니다.

* 메시지 예시: “이 RFP 제안서의 *목차·체크리스트·다음 액션*을 만들어줘. 결과는 목록으로 간결하게.”

* 원하면 Claude/Gemini를 사용해도 동일합니다. (심플100은 **모델 불가지론**)

#### *Step 3\) 응답 가져오기 — ‘상자에 담아 오기’*

* ChatGPT가 만든 **목차/체크리스트/다음 액션**을 그대로 복사해 \*\*심플100 ‘붙여넣기’\*\*에 넣습니다.

* 모델이 **JSON**으로 반환했다면 파일로 저장해 **업로드**해도 됩니다(가능 시). 포맷은 심플100이 알아서 **정리/검증**합니다.

#### *Step 4\) 심플100이 자동으로 ‘정리·꽂기’*

* **목차/요약** → **산출물 카드**(버전/출처 링크 포함)

* **체크리스트** → **체크리스트 카드** 생성 → 항목은 **할 일**로 분해, **담당/마감일** 추천

* **다음 액션** → **일정/할 일**로 플래너에 배치, 간트/타임라인 자동 갱신

* 모든 항목은 원본(RFP·메일)과 \*\*연결선(derived\_from / depends\_on)\*\*으로 추적 가능

#### *Step 5\) 화면에서 바로 보이는 변화*

* 타임라인/간트에 **신규 마일스톤·할 일** 등장, **리스크 신호** 카드 표시

* 대시보드 **다음 액션** 카드 반영, **포스트모템 초안** 자동 시작

**팁**: “결과를 3가지(목차·체크리스트·다음 액션)로 나눠줘”, “각 항목 한 줄”, “요구조건 매핑 포함”.  
**예외 처리**: 들쭉날쭉한 답변은 ’정리 도우미’가 재정렬. 민감 정보는 **라벨링/권한 확인** 후 진행.

**왜 유연한가?**

* 오늘은 ChatGPT, 내일은 Claude/Gemini를 써도 **운영 방식은 그대로**. 결과를 \*\*같은 상자(S100 토큰)\*\*에 담아 오기 때문에 모델 교체/혼용 비용이 사실상 0에 가깝습니다.

## 1\) 활용 시나리오

### A. 영상 제작 에이전시 (촬영/편집)

* **Drop:** 브리프/콘티/계약서 업로드, 메일 커넥터 연동

* **Plan:** 마감일 D 입력 → **역산 엔진**이 촬영·편집·검수·납품 마일스톤/체크리스트 자동 제안(±2\~3일 조정)

* **Loop (핵심 가치 반영):** 스토리보드/카피를 **외부 AI 라우팅(예: ChatGPT/Claude/Gemini 브라우저)** 또는 **내부 AI**로 생성 → **결과 계약**으로 받아 **artifact/checklist/task 토큰** 자동 반영

* **Report:** 납품 후 **포스트모템**(리스크/지연/교훈) 자동 작성→템플릿화

### B. 공공 입찰/제안서

* **Drop:** 공고문/과업지시서/QA/레퍼런스 업로드, 메일 스레드 자동 수집

* **Plan:** 요구조건 기반 **제출 타임라인 & 체크리스트** 자동 생성, 담당 배정

* **Loop (핵심 가치 반영):** 초안/목차/리스크표를 **외부 AI 라우팅(예: ChatGPT 브라우저/Claude/Gemini)** 또는 **내부 AI**로 생성 → **S100 토큰**으로 흡수(버전/감사 로그 유지)

* **Report:** 심사 피드백을 지식화하고 다음 입찰에 **교훈/금지사항** 자동 반영

### C. 교육/행사

* **Drop:** 기획서/연사자료/스폰서 계약/운영 매뉴얼, 회의록 자동 전사·요약

* **Plan:** 일정·리허설·홍보 마일스톤 자동 생성, 의존/자원 충돌 감지

* **Loop (핵심 가치 반영):** 홍보 카피/포스터 문구를 **외부 AI 라우팅(예: ChatGPT/Claude/Gemini 브라우저)** 또는 **내부 AI**로 생성 → 자산/할 일 토큰으로 연결

* **Report:** **체크인·피드백·성과 지표** 집계 및 리포트 자동화

공통 철학: **폴더에 넣으면 끝**(핵심 가치 1). **S100 토큰 표준**으로 **외부 AI를 유연하게** 연결(핵심 가치 2).

## 2\) 핵심 차별화 (Folder‑First vs Platform‑First)

* **(핵심 가치 1\) Folder‑First 무마찰:** 기존 **폴더/메일/드라이브 습관** 유지 → 시스템이 **S100 토큰**으로 표준화·계획·지식화를 자동 수행

* **(핵심 가치 2\) AI‑불가지론 & 유연성:** **S100 토큰 표준 \+ 결과 계약**으로 외부 AI의 **입·출력**을 유연하게 연결(작업별 최적 모델 혼용)

* **Platform‑First(타사 다수) 대비:** 자사 플랫폼으로 **이주** 요구 **없음**, 산출물·업무·지식이 **토큰 그래프**로 끊김 없이 이어짐

## 3\) Why Now

* 생성형 AI 보급으로 **산출물 폭증** ↔ 표준화/거버넌스 **공백**

* 팀은 이미 폴더/메일 중심 → **무마찰 도입** 수요

* **RBAC/감사/보존·파기/DSR** 등 규제·감사 요구 상승 → **토큰 단위 거버넌스** 필요

## 9\) S100 토큰 정의 (요약)

**S100 토큰 \= 공통 헤더 \+ 타입별 페이로드 \+ 관계(그래프) \+ 보안 라벨**로 구성된 **표준 데이터 최소 단위**.  
모든 이메일/파일/회의/업무/산출물/리포트/지식 항목을 **동일 규격**으로 다루고, 토큰 간 링크로 **프로젝트 생애주기**를 구성.

### 9.1 공통 헤더 (예시)

{  
  "s100": {  
    "version": "s100-0.1",  
    "token\_id": "s100:token:task:proj\_001:abc123",  
    "type": "task",  
    "project\_id": "proj\_001",  
    "source": {"connector":"gmail","uri":"mailto:...","ingested\_at":"2025-08-16T01:23:45Z"},  
    "lifecycle": {"state":"planned","created\_at":"2025-08-16T01:23:45Z"},  
    "security": {"labels":\["internal"\],"rbac":{"read":\["pm"\]}},  
    "links": \[{"rel":"depends\_on","to":"s100:token:file:proj\_001:def456"}\]  
  },  
  "payload": {"title":"스토리보드 초안 작성","due":"2025-08-21","deps":\[\]}  
}

### 9.2 대표 타입 (요약)

* **email, file, event, task, artifact, report** 등

### 9.3 라이프사이클

**MINT → ENRICH → PLAN(역산) → EXECUTE → REPORT → ARCHIVE/REDACT** — 토큰 상태 전이로 감사/KPI 가능

## 13\) 경쟁사 대비 **차별화 포인트** (핵심 정리)

1. **(원칙 1\) 무마찰 도입:** 폴더/메일/드라이브 **그대로** → **자동 전처리·계획·업무화**

2. **(원칙 2\) AI‑불가지론:** **S100 토큰 표준 \+ 결과 계약**으로 **외부 브라우저형(ChatGPT/Claude/Gemini) \+ 내부 AI**를 **작업별 유연 혼용**

3. **E2E 생애주기 자동화:** 착수→진행→**포스트모템/지식화**까지 닫는 구조

4. **거버넌스 by Design:** 토큰 단위 **RBAC/감사/보존·파기/DSR**, 감사 가능한 AI 사용

5. **멱등·출처 보장:** 해시/프로비넌스/세그먼트 토큰으로 **중복·혼선 방지**

6. **벤더 락인 회피:** 특정 플랫폼 종속 대신 **표준 토큰**으로 생태계와 **느슨한 결합**

## 14\) 이게 왜 **혁신적**인가

* **기술 혁신:** 산재한 업무 데이터를 **조합 가능한 토큰 그래프**로 표준화하고, LLM 결과까지 **계약 기반**으로 닫는 **Loop‑Closed AI 운영**

* **업무 혁신:** PM의 반복 노동(정리/계획/보고)을 대체 → **속도/일관성/감사 가능성** 동시 확보

* **시장 혁신:** “우리 플랫폼으로 옮기세요”에서 \*\*“당신 폴더에 맞출게요”\*\*로 패러다임 전환

* **경제성:** 도입 장벽↓, 기존 스택 유지, **TCO 절감**과 **ROI 단기 가시화**(파일럿 4–8주 내 KPI 체감)

* **확장성:** 표준 토큰 모델은 **버티컬 템플릿**(영상/교육/행사/공공)로 빠르게 확장·복제 가능

한 줄 요약: **플랫폼 이주 없는 AI 운영 표준** — S100 토큰과 결과 계약으로 **프로젝트가 스스로 굴러갑니다.**
