# 프로젝트 개요 (Project Brief)

## 1. 프로젝트 개요

VibeMemory는 개발자의 GitHub 리포지토리를 단순한 코드 저장소에서 **'살아있는 지능형 지식 자산'**으로 변환하는 시스템입니다.

사용자가 수동으로 '가져오기'한 리포지토리의 모든 `.md` 파일을 분석하여 RAG(검색 증강 생성) 데이터베이스를 구축합니다. GitHub Webhook과 연동하여 push 이벤트를 실시간으로 감지하고, 변경된 파일의 RAG 청크(chunk)만 외과적으로(surgically) 업데이트하여 항상 최신 상태의 지식베이스를 유지합니다.

또한, `projectbrief.md`, `techContext.md` 등 프로젝트 핵심 문서를 AI가 분석하여 자동화된 리뷰를 생성하고, 시스템의 처리 진행률(P0~P5)을 시각적 대시보드로 제공하며, 프로젝트별/전체 챗봇을 제공합니다.

### 핵심 특징
- **자동화된 지식 변환**: GitHub 리포지토리의 마크다운 파일을 자동으로 분석하여 RAG 데이터베이스 구축
- **실시간 동기화**: GitHub Webhook을 통한 push 이벤트 감지 및 변경 파일만 선택적 업데이트
- **AI 기반 분석**: 핵심 문서를 자동 분석하여 아이디어, 기술, 특허 가능성 리뷰 생성
- **지능형 검색**: 하이브리드 검색(FTS + Vector)을 통한 정확한 지식 검색
- **대화형 인터페이스**: 프로젝트별 및 전체 프로젝트 통합 챗봇 제공

## 2. 프로젝트 목표

### 2.1 핵심 문제
개발자들이 GitHub에 축적한 수많은 프로젝트와 문서들은 시간이 지나면서 **"잊혀진 지식 자산"**이 되고 있습니다. 
- 프로젝트의 핵심 아이디어와 기술적 결정을 다시 찾기 어려움
- 문서가 분산되어 있어 전체 맥락 파악이 어려움
- 새로운 팀원이나 협업자가 프로젝트를 이해하는 데 시간이 오래 걸림
- 유사한 문제를 해결한 과거 프로젝트를 찾기 어려움

### 2.2 최종 목표
**GitHub 리포지토리를 지능형 지식 자산으로 자동 변환**하여, 개발자가 언제든지 프로젝트의 핵심을 빠르게 이해하고 활용할 수 있도록 하는 것

### 1.3 기대 효과
- **지식 재사용성 향상**: 과거 프로젝트의 기술 패턴과 해결책을 쉽게 찾아 재사용
- **온보딩 시간 단축**: 새 팀원이 프로젝트를 이해하는 시간을 대폭 단축
- **의사결정 지원**: AI가 프로젝트를 분석하여 아이디어, 기술, 특허 가능성을 제시
- **지속적인 학습**: 프로젝트가 자동으로 최신 상태로 동기화되어 항상 최신 정보 유지

## 3. 핵심 기능

### 2.1 자동 지식 변환
- **P1 인덱싱**: GitHub 리포지토리의 모든 마크다운 파일을 자동으로 수집하고 메타데이터 저장
- **P2 임베딩**: 파일 내용을 의미 단위로 청킹하여 벡터 임베딩으로 변환 (RAG 기반 검색 준비)
- **P3 AI 리뷰**: 핵심 문서를 분석하여 프로젝트의 아이디어, 기술 패턴, 특허 가능성을 자동으로 추출

### 2.2 지능형 검색 및 챗봇
- **하이브리드 검색**: 키워드 검색(FTS)과 의미 검색(Vector)을 결합하여 정확도와 관련성 극대화
- **프로젝트별 챗봇**: 특정 프로젝트의 문서만을 기반으로 질문에 답변
- **전체 프로젝트 챗봇**: 모든 프로젝트를 통합하여 조직 전체의 지식을 검색

### 2.3 실시간 동기화
- **GitHub Webhook**: Push 이벤트를 감지하여 변경된 파일만 자동으로 업데이트
- **릴리즈 노트 자동 생성**: 커밋 메시지를 분석하여 변경 사항을 요약

### 2.4 사용자 경험
- **대시보드**: 모든 프로젝트를 한눈에 보고 관리
- **실시간 진행률**: 프로젝트 임포트 및 스캔 진행 상황을 실시간으로 확인
- **탭 기반 UI**: 아이디어 리뷰, 진행 상황, 챗봇을 탭으로 구분하여 직관적인 탐색

## 4. 타겟 사용자

### 3.1 주요 사용자
- **개발자**: 자신의 프로젝트를 체계적으로 관리하고 과거 작업을 쉽게 찾고 싶은 개발자
- **기술 리더**: 팀의 기술적 결정과 패턴을 문서화하고 공유하고 싶은 리더
- **스타트업/기업**: 조직의 지식 자산을 체계적으로 관리하고 재사용하고 싶은 조직

### 3.2 사용 시나리오
1. **프로젝트 온보딩**: 새 팀원이 프로젝트의 핵심을 빠르게 이해
2. **기술 결정 참고**: 유사한 문제를 해결한 과거 프로젝트의 기술 패턴 참고
3. **지식 검색**: "어떤 프로젝트에서 이 기술을 사용했지?" 같은 질문에 즉시 답변
4. **프로젝트 리뷰**: AI가 프로젝트를 분석하여 개선점이나 특허 가능성 제시

## 5. 핵심 가치 제안

### 4.1 자동화
- 수동 문서화 불필요: GitHub에 이미 있는 문서를 자동으로 활용
- 지속적인 업데이트: Push 이벤트로 자동 동기화되어 항상 최신 상태 유지

### 4.2 지능화
- AI 기반 분석: 단순 검색이 아닌 프로젝트의 핵심을 이해하고 요약
- 의미 기반 검색: 키워드가 아닌 의미로 검색하여 더 정확한 결과

### 4.3 통합화
- 모든 프로젝트를 하나의 플랫폼에서 관리
- 프로젝트 간 지식 공유 및 재사용 용이

## 6. 차별화 포인트

1. **GitHub 네이티브**: GitHub와 완전히 통합되어 별도 문서화 작업 불필요
2. **RAG 기반**: 단순 키워드 검색이 아닌 의미 기반 검색으로 더 정확한 결과
3. **AI 자동 분석**: 프로젝트의 아이디어, 기술, 특허 가능성을 자동으로 추출
4. **실시간 동기화**: Webhook을 통한 자동 업데이트로 항상 최신 정보 유지
